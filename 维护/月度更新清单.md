## 预期时间：15-20 分钟

- 查 5 个网站：~5 分钟
- 更新 2-3 个章节快照：~10 分钟
- 更新 CHANGELOG + 版本号：~2 分钟
- Commit：~1 分钟

---

## 阅读约定 · 页面探索一律走 Jina Reader 代理

> [!IMPORTANT]
> 本清单里所有需要"打开页面/抓页面内容"的动作，**统一通过 [Jina AI Reader](https://jina.ai/reader/) 代理读取**——不要直接抓原站。
>
> **代理格式**：`https://r.jina.ai/<原始URL>`
> **搜索格式**（可选）：`https://s.jina.ai/<查询词>`
>
> **为什么**：
> - 原站经常上 Cloudflare / WAF / JS 渲染，`curl` 和 `WebFetch` 取不到正文；Jina Reader 已统一把页面渲染并转成 LLM 友好的 markdown。
> - 代理返回稳定的纯文本，便于人眼快读和 diff，也便于喂给 AI 做对比。
> - 失效或限流时再退到 `archive.org` 或厂商官方 API（`gh`/RSS/`arXiv`）。
>
> **用法**：浏览器里可以直接打开 `https://r.jina.ai/https://example.com/path`；命令行里 `curl -s https://r.jina.ai/https://example.com/path` 即得 markdown。下文每条 URL 都已按此约定改写。

## 工具辅助（推荐）

### 自动化脚本（不必须，但值得写一次）

你可以写一个 bash + curl 脚本做 A1 的前 3 项——**全部走 Jina Reader 代理**（`r.jina.ai`），避免被原站的 Cloudflare / JS 渲染挡掉：

```bash
#!/bin/bash
# check-month.sh —— 月度检查辅助
# 约定：所有页面探索一律走 https://r.jina.ai/<原 URL>

JINA="https://r.jina.ai"

echo "=== LM Arena ==="
curl -s "$JINA/https://arena.ai/leaderboard" | head -120

echo "=== OpenRouter ==="
curl -s "$JINA/https://openrouter.ai/rankings" | head -80

echo "=== SWE-bench Verified ==="
curl -s "$JINA/https://www.swebench.com/verified.html" | head -80

echo "=== 附录 B 链接存活扫描（通过代理，规避反爬误报） ==="
awk -F'[()]' '/\(http/ {print $2}' 附录/B-参考文献.md | while read url; do
  code=$(curl -s -o /dev/null -w "%{http_code}" "$JINA/$url")
  echo "$code $url"
done
```

> [!TIP]
> Jina Reader 免费额度足够月度用量；如要更高配额或更稳定的吞吐，注册账号拿 API key 后加 `-H "Authorization: Bearer <KEY>"`。书的维护**禁止**把 key commit 进仓库，放 `~/.config/jina/token` 或环境变量。

## 节奏失控时

**3 个月没做月度更新**？别堆着。做一次**季度审查**（见 [季度审查清单](季度审查清单.md)）把积累一起消化。

**6 个月没做**？考虑整体重审。发一个 minor 版本（v1.0 → v1.1）。

**1 年没做**？书的可信度已经严重受损——这时候要么做 major 重构，要么标注 `DEPRECATED`。
