> [!IMPORTANT]
> **已锁定的框架决策（2026-07-14）**——三条扩写轴的形态已定：
>
> 1. **多智能体 / 链→图**：新增**主书第 7 章 · 从链到图：多智能体系统的可靠性**（紧邻现第 6 章后）。这是唯一触发主书重编号的改动。
> 2. **开篇 thesis**：不加"论纲"新章，直接**升级 [前言](../00-前言.md) 的脊梁句**（链路→"并且越来越是一张图；守护的不再是规格偏离，是行为漂移"）。
> 3. **哲学分量**：六条哲学线**织入现有章**，不新开"间奏"或哲学部。
>
> **重编号规则（单一事实源）**：插入新第 7 章后，**原第 7–20 章顺延为第 8–21 章**——**已于 2026-07-15 随第 7 章《从链到图》落位一次性执行**（display-only，文件名与链接路径不变；同步了 [README](../README.md) / `mkdocs.yml` nav / 各文件 H1 / 复习卡 / [漂移度表](漂移度表.md) / 全书"第 N 章"交叉引用，见 §六）。本文其余位置仍保留**重编号前**的旧章号作为规划语境，不逐一回改。
>
> **重编号的两个正则盲区（2026-07-15 补，务必手动 grep 补扫）**：主扫用 `第\s*(\d+)\s*章`，但它抓不到两类形态，`mkdocs build --strict` 也查不出（章号是纯文本、非链接）——
> 1. **无「第」前缀的复合章号**：如"第 5、7 章"里的"7"、"第 12 / 13 / 16 章"。加章后专门 grep `第\s*[0-9]+\s*[、，,和及/]\s*[0-9]+\s*章` 复核每个裸数字。（v1.12.0 就漏了前言"第 5、7 章"，"7"错指向新《从链到图》，直到本轮才补。）
> 2. **计数词与散文枚举**：如"六个基本面……这六章"。插入章会改变某一"部分"的**章数**，要 grep `[六七八]个基本面`、`这[六七八]章` 一类计数词，并回看该枚举是否漏列新章名。（v1.12.0 第二部分 6→7 章，前言与 README 的"六个基本面"都漏 bump。）

## 一、为什么要扩写

更名不是换封面，它改变了这本书的**隐性承诺**：

- 《AI 时代 SRE 架构师**之路**》是一条**路径**——只要对一个读者是完整旅程就够了，允许留白。
- 《AI 时代的**可靠性架构**》是一门**学科的参考系**——它隐含承诺"这是这块领域的地图"。于是原来可接受的留白，变成参考书里显眼的洞。

三股力量叠加，指向"要补"：

1. **书名抬高了下限**：从"够一个人走完"抬到"覆盖这门学科"。
2. **前沿移位**：2025–26 的重心从单模型 / 单链路，移到多智能体、MCP、推理模型、合规。
3. **已有内部背书**：[漂移度表](漂移度表.md) 与深入分区分诊已沉淀出一份经两名独立缺口分析员论证的待写清单，本文把它与前沿、哲学两轴合并重排。

## 六、优先级与写作顺序

1. **合规工程**（第 16 章补丁 + 短新篇）——唯一有外部 deadline（EU AI Act 2026-08），先写。
2. **"从链到图"新章 + 第 4 章扩写**——最大的**概念**洞，撑得起新书名，且解锁后面 MCP / 多智能体 / 护栏的共同语言。
3. **MCP 与工具供应链**——证据最强，[深入 17](../深入/17-LLM网关的SRE视角.md) 已把接口抛出来等着接。
4. **护栏架构 + 思考预算**——补"守"与推理模型区间，工程读者当下最痛。
5. 其余四篇（长任务运行时 / 语音会话 / GPU 机群 / 长上下文并行），按项目暴露的需求补。
6. **哲学织入**贯穿以上，最后统一走一遍，确保六条线各归其位、不"贴上去"。

> [!IMPORTANT]
> 主书插入只剩**一处**触发章节重编号：新第 7 章"从链到图"（§三决策 1）。thesis 走升级前言、哲学走织入现有章，都不动编号；其余工程篇全走**深入新篇 / 扩写**。执行插入时，按顶部"重编号规则"（原第 7–20 章 → 第 8–21 章）一次性同步 [README](../README.md)、`mkdocs.yml` nav、各文件 H1 与 `part-N`、复习卡与全书"第 N 章"交叉引用（见 [样式指南](../样式指南.md) §16）。

> [!WARNING]
> 何时重排这份优先级（自失效触发器，遵 [样式指南](../样式指南.md) §15）：
>
> - EU AI Act 生效时间推迟或豁免范围扩大 → 合规工程降级。
> - 贯穿项目或真实系统先暴露出别的短板 → 让项目定序，不迷信本表。
> - 某个更大的基座模型把护栏 / 思考预算问题"一次冲掉" → 相关篇改为观察，不急写（呼应轴三"苦涩的教训"）。
