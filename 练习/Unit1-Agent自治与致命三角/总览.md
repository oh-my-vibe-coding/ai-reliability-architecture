# Unit 1 · Agent 自治与致命三角（4 周）

> [← 返回目录](../../README.md)  ·  对应知识章节：[第 6 章](../../知识/06-AI自治与上下文架构约束.md)

## 目标
能为"AI 操作生产"类需求画 trifecta 分析图、设计基础设施层约束、划分自治分级。

## 骨架 / 载体

| 骨架（训练意图，永久） | 载体（当期工具，可替换） |
|---|---|
| 致命三角作为威胁模型；为任意输入面做 trifecta 分析 | 当期具体的 prompt injection 攻击样例 / 手法 |
| 设计基础设施层约束（砍三角）、划自治分级 | 沙箱实现（容器 / gVisor / …）、egress 白名单工具、凭证系统 |
| 红队自检的习惯：主动找 ≥3 个漏洞 | 当期 MCP / tool 协议、具体 Agent 框架 |

判断标准看左列：致命三角与"为后果设边界"的思路不随工具变；右列的攻击手法和沙箱实现会持续更新，换了照查即可。

## 产出
对你系统中一个真实场景的**架构约束设计文档**，包含：
- 致命三角分析图（私有数据访问 / 不受信输入 / 外泄通道 的具体实例；工具访问作为放大器一并标出——它加宽入口和出口，但不是独立的一条腿）
- 三套"砍三角"控制方案（沙箱 / egress 白名单 / capability-scoped 凭证）及优劣
- 自治分级表（auto / confirm / forbidden 三档）

**Bloom 目标层级**：**综合 + 评估（Create + Evaluate）** — 设计完整方案并能红队自检。

## Mastery Gate

达到以下标准才算 Unit 1 mastered，可进 Unit 2：

- [ ] Trifecta 分析**覆盖所有输入面**（用户、数据库、外部 API、tool 返回值、MCP……一个都不漏）
- [ ] 3 套方案每套都有**具体的劣势 / 代价**分析（不是"看起来都好"）
- [ ] 分级表的 **Forbidden 档不为空**——必须有些操作禁止 AI 做
- [ ] 红队自检至少发现 **3 个漏洞**，其中**至少 1 个 AI 没想到**
- [ ] 文档至少经过 **2 轮修改**（初稿 → AI 挑错改 → 同事 / 再自己挑改）
- [ ] 能对同事**当场讲清楚**致命三角和自己的防御设计

**未达标的表现**：分析漏了输入面 / 3 套方案只能讲好处讲不出代价 / Forbidden 为空 / 红队全被 AI 想到了。

## 贯穿项目挂接点

Unit 1 让 [🛠️ SRE 事故助手](../贯穿项目-SRE事故助手.md) 从"能跑"变成"能被信任跑"：

| Week | 项目新增 | 验收 |
|---|---|---|
| W1 | `docs/threat-model.md` — 基于项目当前输入面做 Trifecta 分析 | 输入面枚举覆盖 prompt / 日志内容 / 工具返回 / RAG 检出 |
| W2 | `app/permissions.yaml` — 工具权限矩阵（tool × auto/confirm/forbid）| Forbidden 列表非空 |
| W3 | `app/tools.py` 加严白名单 + 自治分级逻辑 | 未授权命令返回 `ERROR: not allowed` |
| W4 | `tests/injection/` — ≥3 个 payload + 红队报告 | 至少 1 个 payload 被基础设施层挡（不靠 LLM） |

**走完 Unit 1，你的项目**：有权限矩阵、有白名单、有 Trifecta 文档、有红队测试。**可以在真实 on-call 场景讨论了**（不再是 toy）。
