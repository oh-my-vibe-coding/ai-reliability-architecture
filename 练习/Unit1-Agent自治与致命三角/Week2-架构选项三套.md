# Unit 1 · Week 2 · 三套控制方案

> [← Unit 1 总览](总览.md)  ·  [← 返回目录](../../README.md)

## 本周目标
为上周识别出的三角，设计 3 套"在基础设施层砍一条腿"的方案，**并讲清楚各自优劣**。

## 阅读 · B3 · 45 分钟（无 AI）
选两篇读：

1. **Anthropic · Agent 架构相关官方文章**（anthropic.com/engineering 下找 "agent" 相关）
2. **OpenAI · Function calling 安全文档**（platform.openai.com 搜 function calling best practices）

重点关注：
- 他们自己设计的**信任边界**在哪（谁是 trusted，谁不是）
- 他们对工具返回值是怎么处理的（尤其是错误处理）

## 产出 · B2 · 60 分钟（不用 AI 写）
为上周识别的场景，设计**三套独立方案**，每套都要能独立降低风险：

### 方案 A · 沙箱隔离
- 具体部署形态？（container / VM / gVisor）
- 网络是什么状态？（默认无 egress？）
- 沙箱的生命周期？（一次会话一个？一个操作一个？）
- **砍掉三角的哪条腿**？
- 代价是什么？（成本、延迟、复杂度）

### 方案 B · 出站白名单 / egress 控制
- 白名单机制在哪层？（L4 iptables / L7 proxy / 应用层 http client）
- 白名单里放什么？怎么维护？
- 如果 Agent 需要调新 API，审批流程？
- **砍掉三角的哪条腿**？
- 代价是什么？

### 方案 C · Capability-scoped 凭证
- Agent 使用的凭证如何分级？（按环境？按操作？按资源？）
- 凭证的过期、轮换、审计？
- 有没有支出上限 / rate limit / blast radius 控制？
- **砍掉三角的哪条腿**？
- 代价是什么？

### 综合对比表
| 维度 | 方案 A | 方案 B | 方案 C |
|---|---|---|---|
| 实施成本 | | | |
| 运维成本 | | | |
| 对开发者友好度 | | | |
| 对已有系统侵入性 | | | |
| 能否被 prompt injection 绕过 | | | |
| 能否组合使用 | | | |

## 写完之后：让 AI 挑错 + 红队
贴给 AI 并问三次：
1. "挑我这三套方案里的技术错误或遗漏"
2. "站在攻击者角度，这三套方案各自最容易的绕过方式是什么"
3. "如果必须只选一套部署，你会选哪套，为什么"

**注意**：第 3 题 AI 给出答案后，你要自己独立想一遍再对比——不要直接照搬。

## 预测 · B1 · 每日 5 分钟
本周每次接触 AI 工具使用场景，猜：
- "这个设计砍了三角哪条腿？"
- "哪条腿最容易被重新接上？"
