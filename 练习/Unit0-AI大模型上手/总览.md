# Unit 0 · AI 大模型上手（2 周）

> [← 返回目录](../../README.md)

## 目标
能直接用 API 从零搭一个"可工具调用、可检索"的最小 Agent。**用过 API 之后，你对上下游的任何 AI 产品都会有更准确的直觉**，后续 5 个单元的架构思考才有根。

## 产出
一个能在终端运行的小工具（<200 行代码），能做三件事：
1. 流式输出
2. 调用 2-3 个本地函数（tool use）
3. 基于一批本地文档回答问题（RAG）

**Bloom 目标层级**：**应用（Apply）** — 能把 API 调用、tool use、embedding 组合成可运行系统。

## Mastery Gate

达到以下标准才算 Unit 0 mastered，可进 Unit 1：

- [ ] CLI 能跑，流式输出正常
- [ ] **至少 2 个 tool** 接入，模型能**自主决定**是否调用
- [ ] RAG 能回答"文档里写了的问题"并**附来源**
- [ ] RAG 能识别"文档里没写的问题"（至少不会胡编）
- [ ] 你能**用自己的话画出** tool use 的两次 API 调用流程
- [ ] 产出了一页"**此 Agent 的 3 种失败模式**"分析

**未达标的表现**：代码能跑但讲不清原理 / tool 只有一个写死的 / RAG 检索不到正确片段。

## 一手材料
- Anthropic API 文档 · messages / tool use / streaming 三章
- Anthropic · 《Building effective agents》
- Simon Willison · Embeddings & semantic search 系列

## 为什么放最前
Unit 1 要设计的"约束 AI 操作生产"架构，前提是你真的用 API 构建过一个能操作的 AI。**没动手构建过，约束设计全是纸上谈兵**。

## 贯穿项目挂接点

**Unit 0 是 [🛠️ 贯穿项目 · SRE 事故助手](../贯穿项目-SRE事故助手.md) 的启动阶段**。

本 Unit 的 Week 任务**不是孤立 demo**，而是搭出项目的最小骨架：

| Week | 项目新增 | 验收 |
|---|---|---|
| W1 | `app/agent.py` + `app/tools.py`（最小 tool use CLI）| 能查"主机磁盘 / 内存" |
| W2 | `app/ingest.py` + `app/retrieve.py` + `data/runbooks/`（≥20 条）| 能基于本地 runbook 回答 5 个事故问题（[smoke eval](../贯穿项目-SRE事故助手.md#unit-0--启动最小系统2-周)） |

**走完 Unit 0，你有**：一个能接受事故问题、检索 runbook、调 shell 工具、给出带来源回答的 CLI Agent。这是 Unit 1-5 的地基。
