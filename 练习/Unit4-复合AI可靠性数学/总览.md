# Unit 4 · 复合 AI 系统可靠性数学（3 周）

> [← 返回目录](../../README.md)  ·  对应知识章节：[第 4 章](../../知识/04-系统架构与复合AI可靠性数学.md)

## 目标
能为任意 Agent 链路算端到端正确率、设计 verifier 插入点、为 step 数做预算。

## 骨架 / 载体

| 骨架（训练意图，永久） | 载体（当期工具，可替换） |
|---|---|
| 端到端正确率的乘积数学；为 step 数做预算 | 当期模型的单步正确率数字 |
| Verifier / Gate 插入点设计；失效放大分析 | 具体 Agent 框架（DSPy / LangGraph / 自建）|
| 加 gate 的成本-收益对比、在评审会上能被质疑且能反驳 | 当期模型定价、延迟、具体编排工具 |

判断标准看左列：乘积数学和 verifier 设计的思路恒久；右列的单步正确率会随模型变好而上升（参见 [第 4 章的自失效触发器](../../知识/04-系统架构与复合AI可靠性数学.md)——单步可靠性逼近 1 时，约束会从"压缩步数"翻转为"每步可验证可回滚"）。

## 产出
对一个现实 Agent 系统的**复合可靠性评审文档**，包含：
- Step 预算分析
- Gate 方案（verifier / retrieval 核验 / citation check 等插入点）
- 失效放大分析

**Bloom 目标层级**：**评估 + 综合（Evaluate + Create）** — 审视现有系统的架构合理性并给出改进方案。

## Mastery Gate

达到以下标准才算 Unit 4 mastered：

- [ ] 选一个**真实（或半真实）Agent 系统**做评审，不是纸上系统
- [ ] 算出**端到端正确率**（乘积计算），和团队对这个数字达成共识
- [ ] Gate 插入点**≥3 处**，每处解释为什么**这里**而不是别处
- [ ] 失效放大分析覆盖**≥2 种级联场景**（前面小错怎么变后面大错）
- [ ] 有**成本-收益对比**：加 gate 的代价 vs 预期收益
- [ ] 评审文档能在**架构评审会议上被质疑且能反驳**

**未达标的表现**：没选真实系统 / 没算端到端数字 / gate 位置拍脑袋 / 只讲加 gate 不讲成本。
