# Unit 3 · LLM 推理 SLO ＋ 静默降级（4 周）

> [← 返回目录](../../README.md)  ·  对应知识章节：[第 5 章](../../知识/05-AI推理服务的可靠性工程.md)

## 目标
写出含 TTFT / 吞吐 / 质量的推理 SLO，解释 GPU × context × QPS 三角约束，设计静默降级检测。

## 骨架 / 载体

| 骨架（训练意图，永久） | 载体（当期工具，可替换） |
|---|---|
| 推理 SLO 三类（延迟 / 容量 / 质量）；为每个 SLI 定目标值 | 当期推理引擎（vLLM / TGI / SGLang）的具体指标名 |
| GPU × context × QPS 三角约束的推导；prefill/decode 分开算 | 当期 GPU 型号、显存数字、KV cache 实现 |
| 静默降级检测机制；灰度 / 回滚 / 影子流量决策树 | 具体监控栈、网关产品、模型版本 |

判断标准看左列：容量三角和 SLO 推导的方法不随硬件变；右列的引擎、卡型、显存数字每季度都要重查（参见 [第 5 章的自失效触发器](../../知识/05-AI推理服务的可靠性工程.md)——若推理范式离开自回归 Transformer，连左列的 SLI 集都要整章重审）。

## 产出
**推理服务 SLO ＋ 容量规划 ＋ 静默降级检测方案**，包含：
- 三类 SLI 指标（延迟 / 容量 / 质量）
- 容量规划图（显存-上下文-QPS 三轴）
- 静默降级检测的方法论
- 灰度 / 回滚 / 影子流量决策树

**Bloom 目标层级**：**综合（Create）** — 容量 + SLO + 故障检测的完整工程方案。

## Mastery Gate

达到以下标准才算 Unit 3 mastered：

- [ ] **每个 SLI 都有**目标值（不是"越高越好"）+ 测量方法
- [ ] **容量算清楚**：能推导"给定 workload 需要几张卡"，不只是抄公式
- [ ] Prefill vs Decode 容量**分开算**，不是合在一起
- [ ] 静默降级检测**有具体机制**（不只是"加监控"），覆盖 ≥3 种症状
- [ ] 决策树的**每个分支都有阈值**（什么情况 rollback、什么情况灰度）
- [ ] 能解释"为什么 QPS 不能作为主要 scaling 指标"

**未达标的表现**：SLI 没目标值 / 容量算的是"总 QPS / 平均 latency" / 静默降级只是一句话带过 / 决策树没阈值。
